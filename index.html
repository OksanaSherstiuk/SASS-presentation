SASS

What is SASS

Sass is a preprocessor scripting language that is interpreted or compiled into Cascading Style Sheets.Which means that, it allows us to use variables, nested rules, mixins, functions, and more, all with a fully CSS-compatible syntax.

Why we use SASS?

As stylesheets are getting larger, more complex, and harder to maintain. This is where a SASS pre-processor can help. SASS lets you use features that do not exist in CSS, like variables, nested rules, mixins, imports, built-in functions, and other stuff.

Structure of SASS Project

Project structure must contain folder named src (the source). In the src folder we have all the files and folders in which we will work: index.html, style.scss, assets folder etc.

Initializing SASS

  1. First thing we need is to install node (if you don't have it). To check the node installation you need to type node --version in the terminal and you will see what version do you have or if you haven't node installed. If you don't have node installed on your computer you have to install it from here and after installation type in the terminal node --version to check it out.
  2. We need to Initialize the project by using npm init in the folder of your project. This command will create a package.json file which includes information about the project like dependencies we are using and scripts we will run. You also can use npm init -y for default setup.
  3. You have to add the following scripts in the package.json file. So we can run them later to use SASS and see the output and create a public folder. 

            "scripts": {
            "build:sass": "sass  --no-source-map src/sass:public/css",
            "copy:html": "copyfiles -u 1 ./src/*.html public",
            "copy": "npm-run-all --parallel copy:*",
            "watch:html": "onchange 'src/*.html' -- npm run copy:html",
            "watch:sass": "sass  --no-source-map --watch src/sass:public/css",
            "watch": "npm-run-all --parallel watch:*",
            "serve": "browser-sync start --server public --files public",
            "start": "npm-run-all copy --parallel watch serve",
            "build": "npm-run-all copy:html build:*",
            "postbuild": "postcss public/css/*.css -u autoprefixer cssnano -r --no-map"
          },
  4. We need these dependencies so the SASS can work properly. 

                "dependencies": {
                "autoprefixer": "^10.4.2",
                "browser-sync": "^2.27.7",
                "copyfiles": "^2.4.1",
                "cssnano": "^5.0.17",
                "npm-run-all": "^4.1.5",
                "onchange": "^7.1.0",
                "postcss-cli": "^9.1.0",
                "sass": "^1.49.8"
              }

Features of SASS

  • Variables and Maps

    Variables are used to store information to be referenced and manipulated later. For example, you can use it for colors, font sizes, screen sizes and many other things. We can declare a variable like this: 

                $primary-color: grey;
            $secondary-color: orange;
        
                h1{
              color: $primary-color;
            }

    Maps

    Also we can use Maps for storing the information in pairs of keys and values, and make it easy to look up a value by its corresponding key 

                $colors(
                  "primary-color": gray,
                  "secondary-color": orange
                   )

  • Nesting

    Nesting is a shortcut to create CSS rules for multiple selectors for a specific property. So, instead of rewriting the same set of properties for the different selectors, we can simply nest selectors inside other selectors. 

                #sidebar {
              position: fixed;
              height: 100%;
      
              ul {
                  list-style-type: none;
                  padding: 0;
      
                  li {
                      background-color: $primary-color;
                      color: $secondary-color;
                  }
                }
              }

  • Mixins

    Mixins allow you to define styles that can be re-used throughout your stylesheet. 

                @mixin list-style {
              margin: 0.5rem;
              padding: 1rem;
              list-style: none;
            }
          
                nav ul {
              @include list-style;
            }
